# Estimating Coefficients
- In practice, B0 and B1 are unknown
- For n observations: (x1,y1), (x2,y2), ..., (xn,yn)
- Let ^yi = ^B0 + ^B1xi be the prediction for Y based on ith val of X
	- i-th residual: ei = yi - y\^i
	- **Residuals:** the differences between observed actual values and predicted values) for each data point, representing the vertical distance from the data point to the fitted regression line.

- **there are number of ways of measuring closeness of fit** (^B0, ^B1)
	- one of techniques is a Gradient Descent:
	- most common is: *Least Square approach*
## Least Square Approach
- It finds the line that minimizes the 
	- **Residual Sum of Squares (RSS)**, which is the sum of the squared differences between the observed `Y` values and the `Y` values predicted by the model
	- will change based on (^B0, ^B1)
- least squares criterion


- for rand vars X,Y we have Y = B0 + B1X + epsilon
	- *epsilon* is noise from N(0, sigma^2) distribution

- By taking expectation values in both sides:
	- E[Y] = E[B0] + E[B1X] + E[epsilon]
		- B0 is a const coefficient, so can be pulled out of expectation
	- = B0 + B1E[X] + 0
		- E[eps]=0 b/c error term is assumed to have mean of 0
	- = B0 + B1E[X]
		- *simplified result* 

### **Covariance of X and Y** (find (^B0, ^B1))
- Cov(X,Y) = Cov(X, B0 + B1X + eps)
- = B0Cov(X,1) + B1Cov(X,X) + Cov(X, eps)
	- *Cov(A, B+C+D)  =  Cov(A,B) + Cov(A,C) + Cov(A,D)*
- = 0 + B1Cov(X,X) + 0
	- constant 1 doesn't change, so cov X,1 = 0
	- cov X,eps = 0 since X and eps are independent
	- any Cov w/ itself is just Var of itself

- = B1Var(x)

- B1 = Cov(X,Y)/ Var(X)
- B0 = E[Y] - B1E[X]

- For samples (x1,y1), (x2,y2), ..., (xn,yn):
- E[X] = xmean 
- E[Y] = ymean
- Var(X) = 1/n SUM(i to n) [xi - xmean]^2
- Cov(X,Y) = 1/n SUM(i to n) [xi - xmean]*[yi - ymean]

## Assessing accuracy of Coefficient estimates
- Population regression line: Y = B0 + B1X + eps
- f(X) = 2 + 3X

- **We want to know how close our estimated coefficients (^B0, ^B1) are to the true population coefficients (B0, B1).**

- This is assessed using:
    - **Standard Errors:** Measure the average amount that the estimate (^B0 or ^B1) differs from the actual value (B0 or B1).
    - **Confidence Intervals:** Provide a range of values within which the true coefficient is likely to fall with a certain level of confidence.
    - **Hypothesis Testing:** To determine if there is a statistically significant relationship between X and Y (e.g., if B1 is significantly different from zero).

## Scenario
- finding population u (mew)
- take means of multiple samples
- the means would fall around the "overall mean" point
	- this should be closer to population mean
	- this is described by CLT:
		- **greater sample sizes lead to the sampling distribution of the sample mean becoming more normal and centered closer to the true population mean (μ).**
		- This implies that our sample mean becomes a more reliable estimate of the population mean with larger samples.

### a. Standard Errors: 
- How far off will that single estimate of ^u be? 
- Var(u^) = SE(^u)^2  =  sigma^2 / n

- How close ^B0 and ^B1 are to true vals of B0 and B1:
	- SE(^B0)^2 = sigma^2 [1/n + [x^-2 / SUM(i to n)[xi - xmean]^2]]
	- SE(^B1)^2 = sigma^2 /  [x^-2 / SUM(i to n)[xi - xmean]^2]

		- sigma^2 is variance of random noise in the data
		- we want to know sigma^2 to calc Residual Standard Error

####  Residual Standard Error (sigma) RSE
- the estimate average distance b/w observed Y values and predicted Y values by the regression line
- **RSE helps assess accuracy of models predictions:**
	- smaller RSE means models predictions are, on avg, closer to actual vals

- in general, sigma^2 = Var(eps) is not known, but can be estimated from data
- RSE = sqrt[ RSS / n-2 ]
	- RSS = e^2_1 + e^2_2 + ... + e^2_n

### c. Hypothesis Testing
- Null hypothesis
	- H0 : There is **no relationship** b/w X and Y
		- H0 : B1 = 0
- Alternative hypothesis
	- Ha: There is **some relationship** b/w X and Y
		- Ha : B1 ≠ 0

- we want B1 to be farther away from 0
	- why?
		- if B1 is far, it suggest strong linear relationship b/w X and Y
		- if B0 is 0, that means no relationship b/w X and Y
- also variance of that var should be small
	- why/how?
		- a small variance (or standard error) for ^B1 means our estimate ^B1 is precise and reliably close to the true B1
- in order to check this, we need t-statistics

#### t-statistics
- t = ^B1-0 / SE(^B1)
- Remember
	- SE(^B1)^2  =  sigma^2 / SUM(i to n) [ xi - xmean ]^2

- **p-value:**
	- Easy to compute the probability of observing any number equal to |t| or larger in absolute value, assuming B1 = 0
	- probability of B1 to be 0 IF null is true
		- **thus, we want low probability if** we want to reject the null hypothesis (i.e., conclude there is a relationship). A low p-value (typically < 0.05) means it's unlikely to observe our data if B1 were truly 0.
		- **if high probability, then** we fail to reject the null hypothesis, suggesting there's not enough evidence to conclude a relationship between X and Y.

### Measure of RSE
- If predictions obtained using model are very close to true outcome values, RSE will be small
- RSE = 3.26 translates to
	- Actual sales in each market deviate from the true regression line by approximately 3,260 units on average
- **doesn't tell if model is good or bad**
- **just tells the error, and how accurate the model is** (a metric)

### R^2 statistics
- **the proportion of variance explained**
- It indicates how well the model fits the observed data
- Always takes a val b/w 0 and 1
	- R^2 = TSS - RSS / TSS = 1 - RSS / TSS
	- TSS = SUM(yi-ymean)^2 is the total sum of squares
