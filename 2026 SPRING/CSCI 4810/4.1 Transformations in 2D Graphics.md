# 3 Transformations: 
- **Translation** and **Rotation** (and *Scaling*)
- Translation:
	- `ctx.translate(centerX,centerY)`
		- changes the canvas
- Rotation:
	- `ctx.rotate(rotationAngle * Math.PI / 180)
		- rotates the canvas
- Scaling:
	- `ctx.scale(sx, sy)`
		- scales from origin

## Cumulative Transformations
- transformations are CUMULATIVE!
- Each transformation builds on the previous one. If you translate twice:

- `ctx.translate(100, 0)` — move right 100
- `ctx.translate(50, 0)` — move right 50 _more_
- Total: 150 pixels right!

  
This is why we need `save()` and `restore()` to isolate transformations.

### The save() / restore() Pattern
- every save() needs a matching restore()!
- to prevent transformations from affecting other objects, wrap them in `save()` and `restore()`:

```javascript
// Save the current state (clean slate)
ctx.save();

// Apply transformations
ctx.translate(100, 100);
ctx.rotate(Math.PI / 4);

// Draw your object
ctx.fillRect(-25, -25, 50, 50);

// Restore — transformations are gone!
ctx.restore();

// Next object is unaffected
ctx.fillRect(200, 50, 50, 50);
```

## Translate : Moving the Origin

-  `ctx.translate(x, y)` moves the origin (0,0) to a new position. After translating, all coordinates are relative to this new origin.

### **How Translate Works:**  
- ctx.translate(tx, ty)  
- New coordinates: (x', y') = (x + tx, y + ty)  
- Drawing at (0, 0) after translate(100, 50) → appears at (100, 50) on screen

### Why Use Translate?
- **Simplify object positioning** – draw at (0,0) and translate to final position
- **Enable rotation around object center** – translate to center, rotate, then draw
- **Create moving objects** – just update the translate values

## Rotate: Spinning Around the Origin
- `ctx.rotate(angle)` rotates the coordinate system around the **current origin**. The angle is in **radians**, not degrees!

### **Radians vs Degrees:**  
- radians = degrees × (Math.PI / 180)  
- 90° = Math.PI / 2 ≈ 1.57 radians  
- 180° = Math.PI ≈ 3.14 radians  
- 360° = 2 × Math.PI ≈ 6.28 radians

- **Common Mistake:** Rotation happens around the _current origin_ (0,0), not the center of your object! To rotate around an object's center, you must **translate first**.

#### The Rotation Pattern
- `ctx.save()` – Save the current state
- `ctx.translate(centerX, centerY)` – Move origin to rotation center
- `ctx.rotate(angle)` – Rotate around the new origin
- Draw your shape **centered at (0, 0)**
- `ctx.restore()` – Restore the original state
